# Өгөгдлийн сангийн удирдлагын бичиг баримт болон нууц үгийн бодлогыг хэрэгжүүлэх

## Асуулт
bcrypt болон хуучин hash алгоритмуудын ялгааг тайлбарлаж, яагаад орчин үеийн системүүд bcrypt ашиглах болсон шалтгааныг бичнэ үү.

## Хариулт 

1. Хуучин hash алгоритмууд (MD5, SHA-1):

	•	Маш хурдан ажилладаг.

	•	Нэг ижил нууц үг үргэлж нэг ижил hash утга үүсгэдэг.

	•	Орчин үед rainbow table болон brute force халдлагад амархан өртдөг.

	•	Давс (salt) ихэнхдээ ашигладаггүй эсвэл хамгаалалт багатай.



2. bcrypt алгоритм:

	•	Нууц үгийг hash хийхдээ salt автоматаар ашигладаг.

	•	Удаан тооцоологдохоор бүтээгдсэн тул brute force халдлага хийхэд маш хэцүү.

	•	Cost factor буюу давталтын тоог нэмэгдүүлж аюулгүй байдлыг өсгөж болдог.

	•	Орчин үеийн системүүдэд нууц үг хамгаалахад илүү найдвартай.


Яагаад орчин үеийн системүүд bcrypt ашигладаг вэ?

	•	Хуучин алгоритмууд хэт хурдан тул хакерууд олон сая нууц үгийг хурдан шалгаж чаддаг.

	•	bcrypt нь удаан ажилладаг, salt ашигладаг тул нууц үгийг эвдэхэд илүү хэцүү болгодог.

	•	Тиймээс аюулгүй байдлыг сайжруулахын тулд орчин үеийн системүүд bcrypt-ийг ашигладаг.


## Жишээ

Жишээ нь password123 гэдэг нууц үгийг MD5 алгоритмаар hash хийвэл үргэлж нэг ижил hash гарна. Хэрвээ хоёр хэрэглэгч ижил нууц үг хэрэглэвэл hash нь мөн адилхан байна. Үүнийг хакерууд rainbow table ашиглан амархан тайлж чадна.



Харин bcrypt алгоритм нь salt ашигладаг. Жишээ нь:

	•	password123 + Xy7! → k9Lm2#

	•	password123 + Pq4$ → 8sT1v@



Энд нууц үг ижил боловч salt өөр учраас hash нь өөр гарч байна. Тиймээс хакер нууц үгийг тайлахад маш хэцүү болдог. Иймээс орчин үеийн системүүд bcrypt ашигладаг.
